# 04 要件定義

プログラミングを使って海洋観測データを可視化するまでの思考の手順を体験します。
ここでは、**何を知りたいのか → どんな図が必要か → そのためにどんなデータをどう処理するか**を順番に整理し、最終的にプログラム全体の流れを考えます。

01〜03 では、

* Python の基本
* 実データを `DataFrame` として読み込む方法
* `DataFrame` の列を使って鉛直プロファイルを描く方法

を見てきました。
この 04 では、それらを組み合わせて、**「複数ステーションのCTDデータをどう可視化するか」**を要件として言葉にしていきます。

---

# 1. 何を知りたいか

まず最初に、「このプログラムで何を知りたいのか」を整理します。
今回知りたいのは、CTD観測から得られた次のようなことです。

* 水温は深さによってどう変化するか
* 塩分は深さによってどう変化するか
* クロロフィルは深さによってどう変化するか
* ステーションごとに違いがあるか

つまり、単に 1 つの CSV を読むだけではなく、**複数の観測地点のデータを並べて比較したい**わけです。

---

# 2. どんな図で可視化するか

知りたいことが決まると、次は「どんな図が必要か」を考えます。
今回は、主に **鉛直プロファイル** と **コンター図** の 2 種類を使います。

## 鉛直プロファイル

鉛直プロファイルは、**あるステーションで、深さによって観測値がどう変わるか**を見る図です。

* 横軸：観測値（水温・塩分・クロロフィルなど）
* 縦軸：水深
* 色：ステーション

たとえば 03_作図入門 で描いたような、こういう図です。

```python
plot_profile(df=df, x_col="temp", title="temperature profile", xlabel="temperature (℃)")
```

この図では、**1本の観測の中で深さ方向に何が起きているか**を見やすくできます。

## コンター図

コンター図は、**複数ステーションを並べて、深さ方向と水平方向の分布を一度に見る**ための図です。

* 横軸：ステーション位置
* 縦軸：水深
* 色：観測値

鉛直プロファイルが「1地点の縦の構造」を見る図だとすれば、コンター図は「**複数地点を並べた断面図**」に近いイメージです。

---

# 3. どんなデータを得たのか

次に、観測データそのものを観察します。
今回の観測データは **CSV (Comma-Separated Values)** で保存されています。
ただし、重要なのは **1ファイル = 1ステーション** という点です。

つまり、

* St1 の水温・塩分・クロロフィルは 1 つの CSV
* St2 のデータは別の CSV
* St3 も別の CSV

という形で、**ステーションごとにファイルが分かれています。**

## 実際のファイルの中身

CTD の CSV を開くと、先頭には機器情報などが入っていて、すぐに表データが始まるわけではありません。

```text
// AAQ-RINKO Series Data Processing Software
// Version 1.12
// File Date:2026/06/21 10:32:51
// Copyright (C) JFE Advantech Co., Ltd.

~中略~

[Item]
深度 [m],水温 [℃],塩分,電導度 [mS/cm],EC25 [μS/cm],密度 [kg/m3],シグマＴ,Chl-Flu. [ppb],Chl-a [μg/l],濁度 [FTU],pH,ORP [mV],DO [%],Weiss-DO [mg/l],光量子 [μmol/(m2*s)],G&G-DO [mg/l],B&K-DO [mg/l],
0.000,24.284,30.405,46.152,46890.3,1020.085,20.085,1.204,1.204,1.334,6.061,167.930,96.321,6.752,519.690,6.771,6.772,
0.100,24.287,30.404,46.152,46887.6,1020.083,20.082,1.249,1.249,1.364,6.062,167.968,96.186,6.745,606.444,6.764,6.764,
0.200,24.278,30.420,46.166,46911.2,1020.098,20.097,1.219,1.219,1.394,6.062,168.006,96.220,6.747,679.274,6.765,6.766,
```

02_データ処理入門で見た通り、表データとして扱いたいのは **`[Item]` より下**です。
そのため、CSVを読むときは

1. `[Item]` の位置を探す
2. そこより上をスキップする

という処理が必要になります。

## 今回使う列

すべての列を使う必要はありません。
今回の可視化で主に使うのは、次の 4 列です。

* `深度 [m]`
* `水温 [℃]`
* `塩分`
* `Chl-a [μg/l]`

つまり、**「まずCSVを読む → 必要な列だけ取り出す」**という前処理が必要になります。

# 4. ステーションとファイルの対応をどう持つか

ここで、複数ステーションをまとめて処理するための設計を考えます。
今回の観測では、若狭湾調査とマガキ調査で、**「横軸に何を置くか」**が少し違います。

* **若狭湾調査**
  ステーションごとの位置の違いを表したいので、横軸には **緯度** を使う

* **マガキ調査**
  養殖場内の測点を並べたいので、横軸には **測点番号** を使う

つまり、ステーションごとに必要なのは、単なるファイルパスだけではありません。
**「そのステーションのCSVはどこにあるか」と「コンター図で横軸に使う値は何か」**を、セットで持っておきたいわけです。

そこで、たとえば次のような辞書を作ります。

```python
WAKASA_DATA_DICT = {
    "St1": (WAKASA_ST1_PATH, ST1_LATITUDE),
    "St2": (WAKASA_ST2_PATH, ST2_LATITUDE),
    "St3": (WAKASA_ST3_PATH, ST3_LATITUDE),
    "St4": (WAKASA_ST4_PATH, ST4_LATITUDE)
}

MAGAKI_DATA_DICT = {
    "St1": (MAGAKI_ST1_PATH, 6),
    "St2": (MAGAKI_ST2_PATH, 5),
    "St3": (MAGAKI_ST3_PATH, 4),
    "St4": (MAGAKI_ST4_PATH, 3),
    "St5": (MAGAKI_ST5_PATH, 2),
    "St6": (MAGAKI_ST6_PATH, 1)
}
```

この辞書の意味は、

* `"St1"` というステーション名に対して
* 対応する CSV ファイルの場所と
* コンター図で横軸に使う値

をまとめて持つ、ということです。

こうしておくと、プログラムは

* St1 の CSV を読む
* St1 という名前を列として追加する
* 緯度や測点番号も列として追加する

という処理を、ステーションごとに繰り返せるようになります。

# 5. データをどう整形するか

ここまでの観察から、作図の前に必要な前処理は次のように整理できます。

1. CSVを読み込む
2. `[Item]` より上をスキップする
3. 必要な列だけ取り出す
4. 列名を扱いやすい名前にする
5. ステーション名を列として追加する
6. コンター図用の横軸情報（緯度や測点番号）を追加する
7. 複数ステーションのデータを結合する

たとえば、結合後の `DataFrame` は次のような列を持っていると便利です。

* `depth`
* `temp`
* `sal`
* `chl`
* `station`
* `x`

ここで `x` は、コンター図の横軸に使う値です。
若狭湾なら緯度、マガキなら測点番号が入ります。

# 8. プログラム全体の流れを考える

最後に、ここまでの要件をもとに、プログラム全体の流れを整理します。

```text
ステーションとファイルの対応を決める
    ↓
各CSVを読み込む
    ↓
必要な列だけ取り出す
    ↓
ステーション情報と横軸情報を付ける
    ↓
複数ステーションのデータを結合する
    ↓
temp / sal / chl について鉛直プロファイルを描く
    ↓
temp / sal / chl についてコンター図を描く
    ↓
図を保存する
```

この流れが見えれば、あとはそれぞれを **どの関数で担当するか** に落とし込めます。

たとえば、

* 1ファイル読む → `load_station_data()`
* 全ステーションをまとめる → `concat_station_data()`
* 鉛直プロファイルを描く → `plot_profile()`
* コンター図を描く → `plot_contour()`

という役割分担が考えられます。

# まとめ

データサイエンスでは、いきなりコードを書くのではなく、

1. 何を知りたいか
2. どんな図が必要か
3. どんなデータがあり、どう対応づけるか
4. 作図の前にどんな前処理が必要か
5. どんな関数の部品が必要か
6. 全体をどんな順序で動かすか

を先に整理します。

プログラムは、その要件を実現するための手段です。
今回のCTD可視化では、02 で学んだ「CSVを読む」、03 で学んだ「図を描く」を部品として使いながら、**複数ステーション・複数変数を扱う全体設計**へと進んでいきます。
