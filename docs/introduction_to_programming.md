```

b = 200

print(a+b)

c = a * b

print(c)
```

数値を演算したり、その結果を変数に代入したりすることができます。

## 関数
```
# 足し算関数を定義する
def tashizan(a, b):
    result = a + b
    return result

# 変数a, bを受け取り、足し算し、result変数に代入してreturn(関数の結果を返す)します

# 定義したtashizan関数を呼び出す

answer1 = tashizan(100, 200)
print(answer1)

answer2 = tashizan(a=1000, b=40)
print(answer2)


# 引き算関数も定義してみる
def hikizan(a, b):
    result = a - b
    return result

# 定義したhikizan関数を呼び出し、その返り値をanswer3変数に代入する
answer3 = hikizan(a=500, b=300)
print(answer3)
```
関数を定義(definition)することで、定義した動作を繰り返し呼び出すことができます。
適切な関数名、変数名を名付けすることで処理の流れが見えやすくなります。

## 処理プロセスを作る
データを処理して可視化するためのプログラムは、長く、複雑になるので、一気に作るのは大変です。  
そのため、**いくつかの処理に分割して作ります。**  
**関数の定義**を学んだのはこのためです。

ここで、観測データを可視化するまでの一連の流れを文章で表した模擬プログラムを作ってみましょう。

まず、データパスを定義します。
```Python

data_path =  "/content/drive/MyDrive/kankyogaku2026/data/wakasa/20260624/202606241429_ASTD152-ALC-R02_0184_142943.Csv"

```

次に、データ読み込み関数を作ります。
```Python

def load_data(data_path):

    print(data_path, "を読み込みました")
```

次に、作図関数を作ります。

```Python

def draw_graph(data_path):

    print(data_path, "のグラフを書きました")
```

最後に、これらの関数をまとめて、一連のメイン処理プロセスを書きます。

```Python

def main_process():
    data_path =  "/content/drive/MyDrive/kankyogaku2026/data/wakasa/20260624/202606241429_ASTD152-ALC-R02_0184_142943.Csv"

    load_data(data_path)

    draw_graph(data_path)

```
メインプロセス関数を呼び出して、実行してみましょう。

```Python

main_process()
```

---

このように、観測データを可視化する、という大きな目的は

- **データパスを定義する**
- **データを読み込む**
- **読み込んだデータを可視化する**
  
という**処理の単位に分割する**ことができます。  
工程を分割することで、必要な作業を具体化し、それぞれの実装方法を考えることができます。    
AIにプログラムを書いてもらう際も、**処理の単位を分割し、それぞれについて具体的に要求を記述することで、正確に意図を伝えることができます。**  
また、既に存在するプログラムを読むときも、関数の名前から、どのような処理工程を担っているかを類推することで、**複雑長大なプログラムの全文を読むのではなく、加工が必要な部分だけに絞って読み出す**ことができます。
