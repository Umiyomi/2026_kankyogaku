# 要件定義の流れ
プログラミングを使って海洋観測データを可視化するまでの思考の手順を体験します。  
どのような目的のために、どのようなデータを処理するのかを、コンピュータの目線に立って観察していきます。

---

## 1. 何を知りたいか

今回知りたいのは、

* 水温は深さによってどう変化するか
* 塩分は深さによってどう変化するか
* クロロフィルは深さによってどう変化するか
* ステーションごとに違いがあるか

---

## 2. どんな図で可視化するか
鉛直プロファイル図とは、コンター図とは、どのような図なのか
### 鉛直プロファイル

* 横軸：観測値
* 縦軸：水深
* 色：ステーション

### コンター図

* 横軸：ステーション
* 縦軸：水深
* 色：観測値

---

## 3. どんなデータを得たのか
観測データはComma-Separated Values(CSV)で保存されている。  
1ステーション毎に1ファイル存在している。
ファイル名を観察してみると、ステーションとファイルの対応関係は...
```
MAGAKI_DATA = {
    St1: 0621090317_AAQ177_SNo0636.csv
    St2: 0621090953_AAQ177_SNo0636.csv"
    St3: 0621092021_AAQ177_SNo0636.csv"
    St4: 0621092708_AAQ177_SNo0636.csv"
    St5: 0621094533_AAQ177_SNo0636.csv"
    St6: 0621095111_AAQ177_SNo0636.csv"
}
```
各ファイルの中身はこうなっている

作図に必要なのは"\\[Item]"と書かれた行から下のデータのようだ。  
今回使うのは特にこのカラム。
```
深度 [m],水温 [℃],塩分,Chl-a [μg/l]
```

---

## 4. データをどう整形するか

作図の前に、

1. CSVを読み込む
2. 必要な列だけ取り出す
3. ステーション情報を付加する
4. 複数ステーションを結合する

という前処理を行う。

---

## 5. どんな図を出力するか

変数ごとに図を分ける。

* temp
* sal
* chl

について、

* 鉛直プロファイル
* コンター図

を作成する。

---

## 6. 処理の流れ

```
CSV読込
    ↓
データ整形
    ↓
ステーション結合
    ↓
図の作成
    ↓
保存
```

---

## まとめ

データサイエンスでは、いきなりコードを書くのではなく、

1. 何を知りたいか
2. どんな図が必要か
3. どんなデータが必要か
4. どう処理するか

を先に決める。

プログラムは、その要件を実現するための手段である。
